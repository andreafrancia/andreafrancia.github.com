1. Introduzione

In questo articolo è spiegato come creare un visualizzatore di file di testo in grado di funzionare anche con file molto grandi (anche di dimensioni di un gigabyte).

Scrivere un visualizzatore quando le dimensioni del file di testo sono ragionevoli è molto semplice, basta caricare tutto il file in memoria e visualizzarlo.

Quando il file è molto grande non è possibile caricare tutto il file in memoria ma conviene invece caricare solo la porzione di file attualmente visualizzata.

2. L'applicazione

L'applicazione sarà avviata con la seguente linea di comando:

java -jar LargeFileViewer <path-to-file>

Dove <path-to-file> è il percorso al file da visualizzare. Il file è un file di testo le cui dimensioni possono essere molto grandi, per esempio un 1Gb.

Appena avviata l'applicazione aprirà una finestra che contiene i seguenti componenti:

  • Una JList dove sarà visualizzato il testo del file.
  • Una barra di scrrimento verticale che l’utente potrà usare per scorrere il contenuto del file file.

Vogliamo che la finestra appaia subito anche se il file è molto grande. Quindi la lettura e l’indicizzazione del file deve avvenire in background.

Vogliamo inoltre che se il file è molto grande l'applicazione non carichi tutto il file in memoria ma che carichi solo la porzione visualizzata attualmente sullo schermo.

3. Soluzione

L'applicazione sarà creata in Java l'interfaccia grafica sarà creata usata usando la libreria Swing.

La finestra principale dell'applicazione contiene un unico componente di tipo JList chiamato fileList.

La classe JList non gestisce solo la visualizzazione della lista e la gestione del modello della lista ad una classe che implementa l'interfaccia ListModel. I metodi più importanti dell'interfaccia ListModel sono quelli descritti di seguito:

  1. public interface ListModel {
  3. Object getElementAt(int index);
  4.     int getSize();
  5. }
Listato 1

Il metodo getSize() restituisce il numero di elementi della lista, nel nostro caso il numero di righe del file.

Il metodo getElementAt() restituisce l'elemento all'indice specificato, nel nostro caso il contenuto della riga del file alla posizione specificata.

Tutto quello che dobbiamo fare è creare un implementazione di questa interfaccia che vada a leggere i dati da un file di testo.

3.1 Indicizzazione

Per evitare di caricare tutto il file in memoria faremo in modo che getElement vada a leggere dal file la riga specificata.

Con la riga 1 apriamo il file in lettura. Usiamo la classe RandomAccessFile perché fornisce il metodo getFilePointer() che restituisce la posizione corrente da cui partirà la prossima lettura.

5. Compilazione ed esecuzione

Per compilare il software è necessario ant e JDK 6.

Dopo aver scompattato l'archivio LargeFileViewer.zip in una directory a piacere (indicata con $SOURCE_DIR), dare i comandi:

cd $SOURCE_DIR
ant jar

Questo dovrebbe creare il file jar come $SOURCE_DIR/dist/LargeFileViewer.jar

Per avviare l'applicazione:

cd dist

java -jar LargeFileViewer.jar <nome-del-file>
